<p>Trivia &amp; Notes</p>
<p>The :not() selector is chainable with more :not() selectors. For example, the following will match all articles except the one with an ID #featured, and then will filter out the articles with a class name .tutorial:</p>
<p>article:not(#featured):not(.tutorial) {<br />
    /* style the articles that match */<br />
}</p>
<p>Just like other pseudo-elements and pseudo-class selectors, :not() can be chained with other pseudo-classes and pseudo-elements. For example, the following will add a “New!” word to list items that do not have a .old class name, using the ::after pseudo-element:</p>
<p>li:not(.old)::after {<br />
    content: "New!";<br />
    color: deepPink;<br />
}</p>

<p>On the Specificity of Selectors</p>
<p>The specificity of the :not() pseudo-class is the specificity of its argument. The :not() pseudo-class does not add to the selector specificity, unlike other pseudo-classes.</p>
<p>The simple selector that :not() takes as an argument can be any of the following:</p>
<p>Type selector (e.g p, span, etc.)Class selector (e.g .element, .sidebar, etc.)ID selector (e.g #header)Pseudo-class selector (e.g :first-child, :last-of-type)</p>
